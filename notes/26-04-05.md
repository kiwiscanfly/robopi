## Hardware wishlist

### [Adafruit 16-Channel PWM / Servo HAT](https://www.adafruit.com/product/2327) — $17.50

A HAT that lets the Pi control up to 16 servos or PWM outputs simultaneously over I2C (only 2 GPIO pins needed). PWM timing is handled by the HAT's onboard chip rather than the Pi's CPU, which eliminates software PWM jitter entirely — a cleaner solution than the `LGPIOFactory` workaround.

| Spec | Value |
| ---- | ----- |
| Channels | 16 |
| Interface | I2C |
| PWM frequency | up to 1.6 kHz |
| Resolution | 12-bit |
| Stackable | up to 62 units (992 servos total) |

Requires light soldering to attach the header. Includes a Python library. Needs a separate 5V power supply for the servos.

## Servo motor

Wired an SG90 servo to GPIO 18 and got it moving using `gpiozero.AngularServo`.

### Wiring

- Red wire → 5V (pin 2)
- Brown wire → GND (pin 6)
- Orange wire (signal) → GPIO 18 (pin 12)

### Pulse widths

The default gpiozero pulse widths don't match the SG90. Override them:

```python
servo = AngularServo(18, min_pulse_width=0.0006, max_pulse_width=0.0023)
```

### Jitter

The servo was jittery when driven with the default software PWM. gpiozero warns about this at runtime:

```text
PWMSoftwareFallback: To reduce servo jitter, use the pigpio pin factory.
```

The fix is to use a hardware-backed pin factory. `pigpio` is the one gpiozero recommends but it is not available on Debian Trixie:

```text
Package 'pigpio' has no installation candidate
```

The alternative is `LGPIOFactory`, which is already installed and working on Trixie:

```python
from gpiozero.pins.lgpio import LGPIOFactory

factory = LGPIOFactory()
servo = AngularServo(18, min_pulse_width=0.0006, max_pulse_width=0.0023, pin_factory=factory)
```

### Detaching on exit

Calling `servo.detach()` in the `finally` block stops sending the PWM signal after the script exits, preventing the servo from buzzing and heating up while trying to hold position.

```python
try:
    # move servo
finally:
    servo.detach()
```
